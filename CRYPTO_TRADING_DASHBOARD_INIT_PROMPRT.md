당신은 지금부터 이 프로젝트의 수석 소프트웨어 엔지니어이자 프로덕트 매니저 역할을 수행합니다. 우리는 본격적인 코드 작성에 앞서, 프로젝트의 기반이 되는 모든 필수 문서를 먼저 체계적으로 정의할 것입니다. 

**[핵심 제약 사항]**
내가 명시적으로 코딩을 시작하라고 지시하기 전까지는 **절대 실제 애플리케이션 코드를 작성하지 마세요.** 오직 프로젝트 분석과 문서화 작업에만 집중해야 합니다.

**[프로젝트 초기 개요]**
* 프로젝트 이름: Real-time Crypto Trading Dashboard (가칭)
* 핵심 목표: 바이낸스/업비트 등의 퍼블릭 웹소켓 API를 브라우저에 직접 연결하여 초당 수십 건의 대용량 실시간 금융 데이터를 DOM 병목 없이 60fps로 렌더링하는 고성능 시각화 엔진 구축. 서버 트래픽 비용 Zero 달성.
* 주요 기능:
  1. 퍼블릭 WebSocket 직접 연동 (호가창, 실시간 체결 내역, 캔들스틱 차트)
  2. Canvas / WebGL 기반의 고성능 차트 렌더링 (DOM 렌더링 최소화)
  3. 사용자가 자유롭게 커스텀 가능한 Drag & Drop 위젯 그리드 시스템
  4. 대용량 데이터 스트림 환경에서의 메모리 누수 방지 및 상태 관리(State Management) 최적화
* 기술 스택: 
  * Frontend: Next.js (또는 React), TypeScript, Zustand/Redux (상태 관리), Lightweight Charts / Canvas API, React Grid Layout
  * Backend & Auth: Supabase (유저별 대시보드 레이아웃 및 관심 종목 저장용)
  * Deployment: Vercel

위 개요를 바탕으로 아래 나열된 마크다운 문서들을 작성해야 합니다. 이 프로젝트는 14년 차 프론트엔드 개발자의 압도적인 기술력을 보여주기 위한 포트폴리오이므로, 대규모 데이터 스트리밍 처리, 브라우저 렌더링 최적화(Reflow/Repaint 방어), 웹소켓 연결 관리 관점이 문서에 깊이 있게 녹아들어야 합니다.
